# 技术报告模板与排版规范

## 输出结构

技术报告必须包含以下六个核心部分：

```markdown
# [论文标题] 技术报告

> **一句话总结**: [核心贡献，不超过50字]

---

## 📋 论文信息

| 项目 | 内容 |
|------|------|
| 标题 | [原标题] |
| arXiv | [ID] |
| 代码 | [链接] |
| 类型 | 基础模型 |

---

## 🎯 核心贡献

[详细贡献列表]

---

## 🔧 技术方案

### 研究背景
### 核心技术
### 训练配置

---

## 📊 实验结果

### 主要性能
### 与基线对比

### 定位
### 技术传承
### 后续影响

---

## 📚 参考资料
```

## 排版规范

### 1. 标题层级

```markdown
# 一级标题（论文标题）
## 二级标题（主要部分）
### 三级标题（子部分）
#### 四级标题（细节）
```

**规范**：
- 一级标题仅用于论文标题
- 二级标题带emoji图标
- 三级标题简洁明了
- 层级不超过四级

### 2. 信息表格

使用表格展示论文信息：

```markdown
## 📋 论文信息

| 项目 | 内容 |
|------|------|
| 标题 | DeepSeek LLM: Scaling Open-Source Language Models |
| arXiv | 2401.02954 |
| 提交时间 | 2024-01-05 |
| 代码仓库 | https://github.com/deepseek-ai/DeepSeek-LLM |
| 论文类型 | 基础模型 |
```

**表格对齐**：
- 第一列左对齐
- 第二列左对齐
- 数字列可右对齐

### 3. 引用块突出

使用引用块突出重要信息：

```markdown
> **核心发现**: 数据质量越高，算力预算应更多分配给模型扩展而非数据扩展。
```

或使用emoji：

```markdown
> 💡 **关键洞察**: 开源模型可以通过科学的scaling方法达到与闭源模型相当的性能。
```

### 4. 数据展示

**关键数字加粗**：

```markdown
DeepSeek 67B在HumanEval上达到 **42.7%** 的准确率，超越LLaMA-2 70B的28.7%。
```

**表格展示对比**：

```markdown
| 任务 | 本方法 | 基线 | 提升 |
|------|--------|------|------|
| HumanEval | **42.7%** | 28.7% | +14.0% |
| GSM8K | **63.4%** | 58.4% | +5.0% |
```

### 5. 列表格式

**有序列表**：用于步骤或排名

```markdown
主要贡献包括：
1. **Scaling Laws研究**: 建立了超参数与计算预算的幂律关系
2. **模型构建**: 训练了7B和67B两个规模的模型
3. **开源贡献**: 发布了完整的训练数据和模型权重
```

**无序列表**：用于特性或要点

```markdown
关键特性：
- 高效的注意力机制
- 可扩展的架构设计
- 完整的开源实现
```

### 6. 分隔线

在主要部分之间使用分隔线：

```markdown
---

## 下一个部分
```

### 7. 技术参数展示

使用代码块或表格：

```markdown
**模型配置**:

| 参数 | 7B模型 | 67B模型 |
|------|--------|---------|
| 层数 | 30 | 95 |
| 隐藏维度 | 4096 | 8192 |
| 注意力头数 | 32 | 64 |
| 训练Token数 | 2.0T | 2.0T |
```

### 8. 公式展示

遵循翻译模板的公式规范：

```markdown
**最优学习率公式**:

$$
\eta_{opt} = 0.3118 \cdot C^{-0.1250}
$$

其中 $C$ 为计算预算。
```

## 各部分规范

### 一、论文信息

必须包含：
- 标题（原文）
- arXiv ID
- 提交时间
- 代码仓库（如有）
- 论文类型分类

**论文类型**：
- 基础模型
- 方法论文
- 应用研究
- 数据集
- 综述

### 二、核心贡献

**必须包含一句话总结**：

```markdown
> **一句话总结**: 首次系统研究开源LLM的Scaling Laws，构建了7B和67B两个规模的高性能开源模型。

主要贡献：

1. **[贡献标题]**: [具体内容，带数字]
2. **[贡献标题]**: [具体内容，带数字]
3. ...
```

**必须有具体参数**：

```markdown
> [背景描述，使用引用块]

### 核心技术

#### [技术点1]

|------|-----|------|
| 层数 | 30/95 | 7B/67B模型 |
| 隐藏维度 | 4096/8192 | |

**核心公式**:

$$
[公式]
$$

| 配置项 | 值 |
```


### 五、技术脉络

**这是最重要的部分**：

```markdown
## 🔗 技术脉络

### 定位

```
[前作] → [本工作] → [后续]
```

本工作是DeepSeek系列的**奠基之作**，为后续所有模型奠定了基础。

### 技术传承

| 技术点 | 来源 | 去向 |
|--------|------|------|
| Multi-step调度器 | 原创 | V2/V3沿用 |
| GQA | LLaMA启发 | MLA前身 |
| 数据质量策略 | 原创 | 指导后续数据工程 |

### 后续影响

1. **为MoE铺垫**: 本工作的基座架构被DeepSeek-MoE继承
2. **为Coder奠基**: 代码能力在DeepSeek-Coder中进一步强化
3. **为V2升级**: 架构升级但仍继承训练范式
```
