# 论文翻译模板与规范

## 输出结构

翻译文件必须包含以下结构：

```markdown
# [论文标题中文翻译]

**[作者列表]**

*[机构名称]*

---

## 摘要

[摘要完整翻译]

---

## 目录

[自动生成的目录结构]

---

## 1. [第一章节标题]

[正文内容]

### 1.1 [子章节标题]

[正文内容]

---

## 参考文献

[如原文有参考文献，保留格式翻译]

---

## 附录

[如原文有附录，完整翻译]
```

## 翻译规范

### 1. 标题翻译

- 主标题、章节标题、子章节标题必须全部翻译
- 保持原文的层级结构
- 数字编号保持不变（如 "2.1" 不变）

### 2. 正文翻译

**核心原则：逐段翻译，禁止概括**

```
❌ 错误示例：
本文提出了一种新方法，该方法在多个任务上表现良好。

✅ 正确示例：
本文提出了一种新方法，该方法包含三个核心组件：（1）基于Transformer的编码器，
（2）多任务联合训练策略，（3）自适应推理机制。实验表明，该方法在文本分类、
命名实体识别和情感分析三个任务上分别达到了92.3%、88.7%和90.1%的准确率。
```

### 3. 公式处理（重要）

#### 3.1 行间公式（Display Formula）

**用于**：独立成行的公式、复杂公式、需要编号的公式

**格式**：使用双美元符号，前后必须有空行

```markdown
学习率的计算公式如下：

$$
\eta_{opt} = 0.3118 \cdot C^{-0.1250}
$$

其中，$C$ 表示计算预算，$\eta_{opt}$ 表示最优学习率。
```

**必须使用行间公式的情况**：
- 包含求和符号 $\sum$
- 包含积分符号 $\int$
- 包含分数 $\frac{}{}$
- 包含矩阵
- 包含多行对齐
- 长度超过20个字符

```markdown
✅ 正确：
损失函数定义如下：

$$
\mathcal{L} = -\sum_{i=1}^{N} \log P(y_i|x_i)
$$

❌ 错误：
损失函数为 $\mathcal{L} = -\sum_{i=1}^{N} \log P(y_i|x_i)$
```

#### 3.2 行内公式（Inline Formula）

**用于**：简短的数学表达式、变量名、简单运算

**格式**：使用单美元符号

```markdown
学习率 $\eta$ 随计算预算增加而减小。

最优批量大小 $B_{opt}$ 与计算预算呈正相关。

模型的隐藏维度为 $d_{model} = 4096$。
```

**行内公式限制**：
- 长度不超过20个字符
- 不包含分数（除非简单分式）
- 不包含求和、积分等大型运算符
- 不包含矩阵

#### 3.3 矩阵和多行公式

**矩阵**：必须使用行间格式

```markdown
注意力权重矩阵为：

$$
W = \begin{bmatrix}
w_{11} & w_{12} & w_{13} \\
w_{21} & w_{22} & w_{23} \\
w_{31} & w_{32} & w_{33}
\end{bmatrix}
$$
```

**多行对齐公式**：

```markdown
优化目标为：

$$
\begin{aligned}
\min_{\theta} \quad & \mathcal{L}(\theta) \\
\text{s.t.} \quad & \|\theta\| \leq C
\end{aligned}
$$
```

#### 3.4 公式编号

如需引用公式，添加编号：

```markdown
$$
\eta_{opt} = 0.3118 \cdot C^{-0.1250} \tag{1}
$$

如公式(1)所示，学习率随计算预算增加而减小。
```

#### 3.5 常见错误

| 错误类型 | 错误示例 | 正确示例 |
|----------|----------|----------|
| 复杂公式强行行内 | $\frac{\partial L}{\partial w}$ | 独立成行 |
| 缺少空行 | 文字$$x$$文字 | 文字\n\n$$x$$\n\n文字 |
| 括号不匹配 | $(x+y$ | $(x+y)$ |
| 上下标错误 | $x_i_j$ | $x_{i,j}$ |

### 4. 表格处理

- 表格标题翻译
- 表头翻译
- 表格内容翻译
- 数值保持不变

```markdown
**表1 | 不同模型的性能比较**

| 模型 | 参数量 | 准确率 | F1分数 |
|------|--------|--------|--------|
| Model A | 7B | 85.2% | 0.831 |
| Model B | 13B | 87.1% | 0.852 |
```

**表格规范**：
- 使用左对齐（默认）
- 数字列右对齐可在表头后加 `------:`
- 表格前后空一行

### 5. 专业术语处理

**首次出现标注英文原文**：

```markdown
我们采用了分组查询注意力（Grouped Query Attention, GQA）机制...
后续提及直接使用中文：分组查询注意力通过减少KV缓存...
```

| 英文 | 中文 | 备注 |

| Transformer | Transformer | 保留英文 |
| Attention | 注意力 | |
| Multi-head Attention | 多头注意力 | |
| Grouped Query Attention | 分组查询注意力 | 首次标注GQA |
| Layer Normalization | 层归一化 | |
| Fine-tuning | 微调 | |
| Pre-training | 预训练 | |
| Inference | 推理 | |
| Embedding | 嵌入 | |
| Token | Token | 保留英文 |
| Benchmark | 基准测试 | |
| Ablation Study | 消融实验 | |
| Scaling Laws | 扩展定律 | |
| Loss Function | 损失函数 | |
| Batch Size | 批量大小 | |

### 6. 列表和枚举

保持原文的列表格式：

```markdown
主要贡献包括：
1. 提出了新的架构设计
2. 在多个基准上取得了最先进的结果
3. 开源了完整的训练代码
```

或使用无序列表：

```markdown
关键特性：
- 高效的注意力机制
- 可扩展的架构设计
- 完整的开源实现
```

```markdown
**图1 | 模型架构示意图**

图中展示了模型的主要组件，包括编码器、解码器和注意力机制。
```

```markdown

[1] Vaswani, A. et al. "Attention is All You Need." NeurIPS 2017.
```

## 分块翻译规范

当论文过长需要分块翻译时：

### 术语表维护

每个分块翻译前，注入当前术语表：

```json
{
  "Transformer": "Transformer",
  "attention": "注意力",
  "GQA": "分组查询注意力 (GQA)"
}
```

翻译过程中发现新术语，更新术语表。

### 分块合并

确保：
- 章节标题不重复
- 术语翻译一致
- 公式编号连续
- 表格完整

---

## 质量检查清单

翻译完成后，自查以下项目：

- [ ] 所有章节标题已翻译
- [ ] 摘要完整翻译
- [ ] 没有遗漏的段落
- [ ] 公式格式正确（行间/行内区分）
- [ ] 公式前后有空行
- [ ] 表格完整翻译
- [ ] 专业术语首次出现标注英文
- [ ] 数字和单位正确
- [ ] 文中没有大段未翻译的英文
- [ ] 术语全文一致
